# OS Tuning Reference


| Category | Optimization | Notes & Impact | Tradeoffs / Caveats |
|---|---|---|---|
| **CPU Isolation & Power Mgmt** | Disable hyper-threading (SMT) | Siblings share L1/L2, branch predictor, and execution ports — a noisy sibling evicts cache lines of the latency-critical thread; disabling dedicates all physical resources to one thread per core. (`echo off > /sys/devices/system/cpu/smt/control`) | Halves logical core count; size the instance accordingly. |
| | Disable C-states and P-states | C1+ exits cost 10–200 µs and P-state transitions introduce 10–50 µs of unstable frequency — both eliminated so the core is at full clock the instant a packet arrives. (`intel_idle.max_cstate=0 processor.max_cstate=0 intel_pstate=disable`) For the most aggressive idle elimination, add `idle=poll` to keep the core in a tight spin-wait loop with zero wake-up latency. | `idle=poll` consumes 100 % CPU per idle core and increases power/thermal draw; `intel_idle.max_cstate=0` disables the `intel_idle` driver entirely and falls back to `acpi_idle` — verify with `cat /sys/devices/system/cpu/cpuidle/current_driver`. |
| | Set CPU governor to `performance` | Pins cores at maximum frequency, preventing the cpufreq governor from downclocking during brief idle gaps even when C/P-states are nominally disabled. (`cpupower frequency-set -g performance`) | |
| | Add `skew_tick=1` | Staggers the jiffies timer tick across housekeeping cores so they don't all fire simultaneously, reducing lock contention and jitter on shared kernel data structures. | Minimal; should be standard on multi-core latency builds. |
| | Isolate application cores | `isolcpus` removes cores from the scheduler pool; `nohz_full` kills the 1 kHz tick on them; `rcu_nocbs` offloads RCU callbacks to housekeeping cores — together making isolated cores interrupt-free except for the application itself. Consider adding `rcu_nocb_poll` so the RCU offload thread polls instead of relying on its own wakeup interrupts, preventing stray IPIs on housekeeping cores from disturbing isolated cores. | `rcu_nocb_poll` keeps the RCU kthread at 100 % on its housekeeping core. |
| | Move IRQs, workqueues, and RCU callbacks to housekeeping cores | Without explicit affinity pinning via `/proc/irq/*/smp_affinity_list` and workqueue cpumask, the kernel routes IRQs and kworker threads onto isolated cores regardless of `isolcpus` — IRQ and workqueue affinity is independent of the scheduler domain. | |
| | Pin application with `taskset` / cgroups | Use `taskset -c` or `cpuset` cgroups to bind the latency-critical process to specific NUMA-local isolated cores. Without explicit pinning, the scheduler may migrate the process on startup before `isolcpus` fully takes effect, or child threads may escape to non-isolated cores. | Requires knowing exact core-to-NUMA topology; use `lscpu --extended` and `numactl --hardware`. |
| **Network Stack** | Enable busy polling | Spin-polls the NIC RX queue for the configured µs window before blocking on an interrupt, eliminating the 5–15 µs interrupt delivery path for sockets that receive traffic within that window. Set `net.core.busy_poll=50` and `net.core.busy_read=50` for blocking sockets; for non-blocking sockets, set `SO_BUSY_POLL` per-socket. Note: `busy_poll` alone does not activate polling on blocking `recv()` — `busy_read` is required for that path. | Burns CPU on the polling core for the configured window; size the µs value to your expected inter-packet gap. |
| | Disable IRQ balancer | `irqbalance` periodically rewrites `/proc/irq/*/smp_affinity`, silently re-routing NIC interrupts onto isolated application cores and undoing all manual affinity pinning. (`systemctl disable --now irqbalance`) | Manual IRQ affinity management required; script it in startup to survive reboots. |
| | Configure RSS, RPS, and XPS | RSS (hardware RX hashing), RPS (software RX steering), and XPS (TX steering) must all target housekeeping cores — not isolated application cores — so NIC processing never preempts the hot path. | |
| | Increase kernel network buffers | Raising `rmem_max`, `wmem_max`, and `netdev_max_backlog` absorbs inbound bursts without drops, preventing tail-latency spikes from packet loss and retransmission when the socket buffer fills. | Over-sized buffers increase worst-case bufferbloat latency; tune to your burst profile, not arbitrary large values. |
| **ENA-Specific Tuning** | Disable GRO, LRO, TSO, and GSO offloads | GRO/LRO coalesce inbound packets before delivery and TSO/GSO batch outbound segments — all add deterministic batching latency in exchange for throughput, unacceptable on the critical path. LRO is typically off by default on ENA but should be verified explicitly. (`ethtool -K eth0 gro off lro off tso off gso off`) | Disabling offloads increases per-packet CPU cost and reduces bulk throughput; only disable on the latency-critical interface. |
| | Disable Dynamic Interrupt Moderation (DIM) | DIM adaptively increases `rx-usecs` under load to batch interrupts; disabling delivers an interrupt on every received frame (or nearly so), removing variable per-packet coalescing delay. (`ethtool -C eth0 adaptive-rx off rx-usecs 0 tx-usecs 0`) | `rx-usecs=0` generates one interrupt per frame and can saturate a core under burst traffic. A common production compromise is `rx-usecs=1` or `rx-usecs=2`, which bounds worst-case coalescing to 1–2 µs while dramatically reducing interrupt load. Profile under peak traffic before committing to 0. |
| | Increase ring buffer sizes | Larger RX/TX rings prevent descriptor exhaustion during µs-scale traffic bursts, eliminating tail-latency spikes caused by drops and retransmission when the ring fills. (`ethtool -G eth0 rx 4096 tx 4096`) | Larger rings consume more memory and can increase average latency by allowing deeper buffering; balance against your burst depth. |
| | Consider DPDK or AF_XDP zero-copy | DPDK's poll-mode driver bypasses the kernel stack saving 1–10 µs per packet; AF_XDP delivers zero-copy from NIC RX ring to user-space buffers at lower integration cost and works on virtualised Nitro with ENA driver support. | AF_XDP zero-copy on ENA requires both a sufficiently recent ENA driver (≥ v2.13.0 as a baseline) **and** a compatible kernel version; verify the exact combination against your AMI/kernel. DPDK requires dedicated cores and NIC ownership, making the interface unavailable to the kernel. |
| | Use ntuple flow steering | Hardware 5-tuple filters steer a trading session's packets to a single RX queue pinned to one core, keeping that core's caches warm and avoiding the cross-core cache migration that RSS's hash distribution can cause. (Nitro v5+; `ethtool -U`) | Limited number of hardware filters; plan filter slots across all sessions. |
| **Memory** | Disable THP on x86; `madvise` on Graviton | On x86, `khugepaged` compaction stalls threads 100 µs–1 ms during page promotion; on Graviton, THP with `MADV_HUGEPAGE` improves TLB reach without compaction jitter, making `madvise` mode correct there. (`echo madvise > /sys/kernel/mm/transparent_hugepage/enabled` on Graviton; `echo never` on x86.) | |
| | Pre-allocate explicit huge pages | 2 MB pages reduce TLB pressure up to 512× for the same footprint, eliminating TLB miss penalties on large working sets; boot-time pre-allocation (`vm.nr_hugepages=N`) avoids runtime page-fault latency and fragmentation. | Huge pages are not swappable and are reserved at boot; over-allocating wastes RAM. Size `N` to your application's working set. |
| | Lock memory with `mlock` / `mlockall` | Prevents the kernel from paging out hot application memory under memory pressure, eliminating major page-fault latency spikes on the critical path. | Locked memory counts against `RLIMIT_MEMLOCK`; ensure limits are raised. Combined with `vm.swappiness=0` to further discourage swap activity. |
| | Disable NUMA balancing | The kernel's auto-balancer migrates pages between NUMA nodes, stalling threads and issuing TLB shootdowns; with explicit `numactl --cpunodebind --membind` placement it adds only jitter with no benefit. (`kernel.numa_balancing=0`) | Requires correct manual NUMA placement; misplacing memory on a remote node with balancing disabled permanently degrades bandwidth. |
| | Set `vm.swappiness=0` | Minimises the kernel's preference for swapping anonymous pages to disk, keeping hot application pages resident in RAM. Combined with `mlock` for belt-and-suspenders protection. | Does not fully disable swap; use `swapoff -a` if zero swap activity is required and RAM headroom is sufficient. |
| **I/O** | Set `none` (noop) scheduler for NVMe | Passes I/O directly to the NVMe driver queue with zero kernel scheduling overhead; appropriate for dedicated NVMe where the device's internal scheduler handles ordering. For shared block devices, fall back to `mq-deadline` which bounds worst-case I/O latency via per-request deadline expiry. | `none` provides no fairness guarantees; only use on dedicated, non-shared storage. |
| | Disable block-layer statistics and entropy collection | Per-request `iostats` and `add_random` execute on every block I/O completion in the interrupt path shared with latency-critical threads; disabling removes unnecessary work from that path. (`echo 0 > /sys/block/<dev>/queue/iostats`, `echo 0 > /sys/block/<dev>/queue/add_random`) | Disabling `iostats` removes `iotop` / `iostat` visibility; re-enable temporarily for debugging. |
